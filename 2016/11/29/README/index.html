介绍

  • 是一个类型与SVN的版本控制系统VCA（Version Control System）
    • GIT是分布式版本控制系统
    • SVN是集中式版本控制系统
    • 相同点：都是版本控制系统
      • 历史记录
      • 多人协作
  • 2005年推出，最初为了更好地管理linux内核而生
  • 关于名字：Git：饭桶，无用的人
    • Angular：有角的，生硬的
  • 作者：林纳斯·脱瓦兹(Linus Torvalds)
    • Linux之父

Git的诞生

  • 关于Git的作者：Linus Torvalds
    • 创造了 Linux
    • 创造了开源世界
    • Ubuntu Linux学习资源
  • 2002年，Linus·Torvalds使用Bitkeeper作为Linus项目的版本控制系统软件
  • Linus·Torvalds 2005年，短短几个星期之内就诞生了Git的最初版本
  • 真正的大规模被开发人员使用大概在2013年左右
    • 2013年之前大部分都是SVN
    • 目前绝大多数，百分之70-80的公司都在使用Git作为自己的版本控制系统

  • 通过在命令太输入git --version查看是否有git环境

命令的分类

  • 控制台自带命令

    • 工具型
      • ping
      • config
      • shutdown
      • winver 查看Windows版本信息
      • ……
    • 文件系统型
      • cd
      • pwd
      • ls
      • mkdir
      • cp

  • CLI(Command Line Interface)程序：命令行应用程序

    • 没有图形界面，直接在终端中运行，由第三方应用程序执行
      • 接受用户通过终端输入的字符
      • 处理执行用户通过终端输入的字符
      • 终端会对你输入的相应的参数或者命令解析执行实现相应的功能
      • 如果有提示信息，直接将提示信息或者说操作信息，输出到终端
  • GUI程序：具有图形界面应用程序
    • feiq

使用终端操作文件系统

`pwd`(print working direction): 打印文件目录。git bash和 powershell可用。
`ls`(list files):列出当前目录下的所有文件。powershell and git bash可用
    * ls可以指定参数
    * ../：返回当前目录的上一级目录
    * ./：当前目录
    * ls -a: 打开文件目录并显示隐藏文件
`cls`: 清屏 cmd and powershell可用
`dir`: 显示当前目录下的所有文件。cmd and powershell可用
`cd`: 切换工作目录。三个终端通用
    + 在cmd或powershell中切盘符：`盘符：`回车即可
`mv`: 移动文件目录
`cp`: 复制目录。在复制时，不会复制子文件和子目录
    * cp `原路径 目标路径`
`exit`: 退出命令窗口
`clear`: 清屏  git bash可用
`mkdir`: 新建文件
`rm`: 删文件或者目录
    * rm -rf 目录名称：可以删除非空目录
    * 在cmd、powershell不支持
`rmdir`: 删目录。只能删除空目录
上键：查看历史命令
touch: 新建文件
cat: 查看指定文本文件
  • powershell: 加强版的cmd。windows XP 以后增加的功能

基本操作

以后所有的项目文件，都使用 utf-8 编码。如果文件是GBK，使用git bash提交时，应该可以自动转码然后提交

原理模型

工作区、暂存区、本地仓库

  • 暂存区起到两个作用：
    • 可以分批次的进行提交
      假设你很长时间没有做过版本记录提交，例如你修复了某一个功能模块的Bug、开发了某个功能模块、修改了某些功能模块代码，这个时候就可以利用暂存区分批次的进行提交。
    • 可以作为一个临时版本存储
      例如可以讲暂存区指定文件恢复到工作区

本地仓库就是最终用来存储历史版本的地方，每一次commit都会醒成一个历史版本保存到本地仓库中
